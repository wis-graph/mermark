# 파일 탐색기 기능 스펙 (표준 관례 기반)

> mermark 좌측 사이드바 파일 탐색기가 가져야 할 **표준 기능 전수 정의**. VSCode Explorer · Obsidian File Explorer · macOS Finder · Sublime 사이드바 + WAI-ARIA Tree View(APG) 리서치로 도출. 이 문서로 기능을 확정한 뒤 **하나씩(페이즈별) 구현**한다.
>
> 정체성: mermark = **반볼트 단일 폴더 에디터**. 루트 = CLI로 연 문서의 폴더. 볼트/멀티루트/콘텐츠검색/그래프는 정체성 밖(→ Obsidian 영역). 탐색기는 "지금 이 문서 주변을 표준적으로 둘러보고 열기".

---

## 0. 현재 구현(`5fb8b0d`)의 문제 — 재작업 방향

방금 랜딩한 탐색기는 **표준에서 벗어났다**. 정정:

| 항목 | 현재(잘못) | 표준(정정) | 근거 |
|------|-----------|-----------|------|
| 위치 | 푸터 토글 **플로팅 팝오버** | **좌측 사이드바**(폭 드래그·토글) | VSCode/Obsidian/Sublime 전부 좌측 사이드바 |
| 폴더 펼침 | **hover 자동 열림**(120ms) | **클릭으로만** 펼침 | 3개 도구 전부 hover 없음 + WCAG 1.4.13(hover 상태변화 안티패턴)·키보드 재현불가 |
| 키보드 | 없음 | ARIA tree 키보드 셋 | APG Tree View 표준 |

`list_dir` 백엔드(read-only 디렉토리 나열)는 **재사용 가능**(정정 대상 아님). 프론트 패널을 사이드바+클릭펼침으로 재구축.

## 1. 사이드바 셸

| 기능 | 동작 | 우선순위 |
|------|------|---------|
| **위치 = 좌측** | 에디터 왼쪽에 세로 패널. 상태바 토글이 아니라 레이아웃 사이드바 | P0 |
| **토글 표시/숨김** | 버튼(상태바 좌 네비 그룹) + 단축키(예 ⌘\ 또는 ⌘⇧E). 접힘이 기본일지 펼침이 기본일지 결정 | P0 |
| **폭 조절** | 사이드바-에디터 경계 드래그(sash). 최소/최대 폭. 폭 영속 | P1 |
| **접힘 상태 영속** | 사이드바 열림/폭이 세션 저장 | P1 |
| **뷰 타이틀 액션** | 상단 바: 새 파일·새 폴더·새로고침·전체 접기 버튼 | P2 |

## 2. 트리 상호작용 (핵심 — hover 금지)

| 기능 | 동작 | 우선순위 |
|------|------|---------|
| **폴더 펼침/접힘 = 클릭** | 폴더 행 클릭 또는 chevron(▶/▼) 클릭이 토글. **hover 자동열림 절대 금지** | P0 |
| **chevron 회전** | ▶(닫힘)→▼(열림), 150–200ms, `prefers-reduced-motion` 존중, `aria-hidden` | P1 |
| **들여쓰기 가이드** | 레벨당 일정 padding-left, `aria-level`과 일치. 선택적 가이드 라인 | P1 |
| **폴더 내용 로드** | 펼칠 때 `list_dir` 호출(클릭 시점 — hover 아님), 한 번 로드 후 캐시. 재-펼침은 캐시 | P0 |
| **compact folders**(선택) | 단일자식 폴더 체인 축약(`a/b/c`) | P3 |

## 3. 키보드 네비게이션 (ARIA Tree — APG 표준)

**roving tabindex**: 트리 전체가 tab stop 1개, 활성 treeitem만 `tabindex=0`, 화살표로 내부 이동.

| 키 | 동작 | 우선순위 |
|----|------|---------|
| **↓ / ↑** | 다음/이전 *노출* 노드로 **포커스** 이동(열고닫지 않음) | P0 |
| **→** | 닫힌 폴더=열기 / 열린 폴더=첫 자식 / 파일=무동작 | P0 |
| **←** | 열린 폴더=닫기 / 그 외=부모로 이동 | P0 |
| **Enter** | 노드 활성화 = **파일 열기**(폴더면 토글) | P0 |
| **Home / End** | 첫 / 마지막 노출 노드 | P1 |
| **타입어헤드** | 글자 입력→접두 매칭 노드로 포커스(파일목록 사실상 필수) | P1 |
| **Space** | (싱글선택) 포커스 노드 선택 / (다중) 선택 토글 | P2 |
| **PgUp/PgDn** | 페이지 이동 | P2 |
| **`*`** | 형제 전체 펼침 | P3 |

## 4. 포커스 · 선택 (분리가 핵심)

| 개념 | 정의 | 우선순위 |
|------|------|---------|
| **포커스 ≠ 선택** | 포커스=키보드 위치(1개), 선택=액션 대상 집합. **화살표 탐색만으론 파일 안 열림**, Enter/더블클릭으로 명시 활성화 | P0 |
| **시각 분리** | 포커스=아웃라인 링, 선택=배경 채움. 겹칠 수 있으니 다른 스타일 | P0 |
| **활성 파일 강조** | 에디터에서 열린 파일 행을 `aria-current` + 좌측 accent 바로 표시(선택과 또 별개) | P1 |
| **단일 선택** | 클릭=포커스+선택 1개 | P0 |
| **다중 선택**(선택) | ⌘/Ctrl-클릭 토글, Shift-클릭 범위, Shift+화살표 | P3 (파일작업 있을 때만 의미) |

## 5. 열기 동작 (에디터 관례)

| 기능 | 동작 | 우선순위 |
|------|------|---------|
| **파일 클릭 → 현재 창 열기** | `.md` 단일클릭 → `openInWindow` re-mount(현재 문서 교체, Obsidian식). 신규 열기코드 재사용 | P0 |
| **미저장 커밋** | 열기 전 `commitBeforeSwitch`(현 문서 미저장분 보존) | P0 |
| **비-md 처리** | 회색 표시 + 클릭 no-op(또는 OS 기본앱 — 정체성 판정) | P1 |
| **프리뷰 vs 고정**(선택) | VSCode/Sublime식 단일클릭=프리뷰(이탤릭)·더블클릭=고정 — mermark 단일창이라 큰 의미 없을 수 있음 | P3 |

## 8. 파일 작업 (쓰기 — 정체성 판정 필요)

> ⚠️ create/rename/delete/move는 **쓰기 IPC 신규**(백엔드 표면 확대 + 위험). mermark이 "네비게이터"인지 "파일 매니저"인지 결정에 달림. Obsidian은 있음, 최소 에디터는 없을 수도.

| 기능 | 트리거 | 우선순위 |
|------|--------|---------|
| 새 파일 / 새 폴더 | 상단 버튼 + 우클릭 | P2(정체성 승인 시) |
| 이름변경 | F2 / 우클릭(에디터라 Enter=열기 유지, Finder식 Enter=rename 안 씀) | P2 |
| 삭제 | Delete + 확인 다이얼로그(휴지통 권장) | P2 |
| 이동(드래그앤드롭) | 폴더로 드래그 | P3 |
| 우클릭 컨텍스트 메뉴 | 새파일/폴더·이름변경·삭제·이동·Finder에서 보기·경로 복사 | P2 |

## 9. 루트 · 네비게이션 (결정됨)

| 기능 | 동작 | 우선순위 |
|------|------|---------|
| **루트 = 연 폴더 (고정)** | CLI로 연 문서의 baseDir가 트리 루트(볼트 없음). 고정 루트 | P0 |
| **상위 이동 `..` 허용** | 최상단 `..`로 상위 폴더 이동 가능(클릭/키보드로만, read-only). ✅ 결정: 고정 루트 + `..` 상위 허용 | P1 |
| **문서 전환 시 루트 리셋** | 새 문서 열면 루트를 그 baseDir로 리셋(ephemeral) | P1 |

## 10. 접근성 (WAI-ARIA Tree, 필수)

- `role=tree`(+`aria-label="파일 탐색기"`) → `treeitem`(폴더엔 `aria-expanded`, 파일엔 없음) → 자식 `role=group`.
- 지연로딩/가상화면 `aria-level`·`aria-setsize`·`aria-posinset` 필수.
- roving tabindex, 포커스 링(`focus-visible`), 스크린리더 발화(이름·펼침상태·레벨).
- **hover 금지**(WCAG 1.4.13). 모든 동작이 키보드로 재현 가능해야.

## 11. 검색 / 필터

- 트리 인라인 파일명 필터는 **선택**(P3). Obsidian은 인라인 필터 없이 Quick Switcher(⌘O)로 분리 — mermark도 파일명 점프는 별도 팔레트가 정체성에 맞을 수 있음.
- 콘텐츠 검색(⌘⇧F)은 **정체성 밖**(반볼트).

## 페이즈별 구현 로드맵 (하나씩)

**P0 — MVP 네비게이터** (표준 최소, 현 잘못 정정):
1. 좌측 사이드바 셸 + 토글(§1).
2. 클릭 펼침 트리 + `list_dir` 클릭 로드(§2, hover 제거).
3. ARIA tree 키보드 ↑↓→←/Enter + roving tabindex(§3).
4. 포커스≠선택 + 단일선택(§4).
5. 파일 클릭→현재 창 열기(§5, 기존 openInWindow 재사용).
6. 폴더 우선(§6, backend 재사용) + 루트=연 폴더(§9).
7. ARIA roles 기본(§10).

**P1 — 완성도**: chevron 회전·들여쓰기(§2), Home/End·타입어헤드(§3), 활성파일 강조(§4), 비-md 처리(§5), 파일 아이콘·truncate(§6), 펼침상태 영속(§7), `..` 상위(§9).

**P2 — 표시·작업**: 정렬 옵션·숨김 토글(§6), 파일 작업(새파일/폴더·이름변경·삭제·컨텍스트메뉴)(§8) — **정체성 승인 후**.

**P3 — 고급**: 다중선택(§4)·드래그이동(§8)·compact folders·프리뷰탭·인라인 필터·외부변경 반영.

## 결정 기록
- ✅ **루트 모델**: 연 폴더 **고정 루트 + `..` 상위 허용**(read-only, 클릭/키보드). 문서 전환 시 루트 리셋(ephemeral). (§9)
- ⏳ **정체성 (§8 파일작업)**: 로딩/메모리 비용은 **≈0**(파일작업은 on-demand IPC, 영속 인덱스/와처 없음 — 부팅·메모리 무관). 진짜 비용은 **위험(데이터 손실→휴지통+확인 필수)·구현 복잡도**뿐. → 사용자 최종 판정 대기(네비게이터 / +기본작업 / 풀매니저).
- ⏳ **다중선택**: 파일작업 없으면 단일선택으로 충분(§4 P3 보류).
- ⏳ **토글 단축키**·기본 열림 상태: 구현 착수 시 확정.

## 구현 방침
스펙 확정(이 문서) → **하나씩(P0→P3) 구현**. P0 MVP는 정체성 판정과 무관하게 read-only 네비게이터라 먼저 착수 가능(현 잘못된 플로팅+hover를 좌측 사이드바+클릭펼침으로 정정). 파일작업(§8/P2)은 정체성 승인 후.
