## [0.9.5] - 2026-07-20

### Fixed

- **CSV의 한글이 깨져 보이던 문제**: `카테고리`가 `ì¹´í…Œê³ ë¦¬`처럼 보였다. BOM(파일 앞머리의 인코딩 표식)이 없는 UTF-8 CSV를 표 라이브러리가 옛 서양 인코딩으로 읽어버린 탓이다. 이제 CSV는 **글자로 먼저 해석한 뒤** 표로 넘긴다 — UTF-8을 엄격하게 시도하고, 그게 아닐 때만 CP949(EUC-KR)로 되짚는다.
  - 이로써 0.9.3에 적어둔 **"CP949 CSV는 깨져 보인다"는 한계도 함께 사라졌다.** 두 경우가 한 규칙으로 닫혔다. (0.9.3에서 "UTF-8은 괜찮다"고 적었던 건 확인한 파일이 전부 BOM 있는 것이었기 때문이다 — BOM 없는 UTF-8이 사각지대였다.)

### Changed

- **표가 읽기 쉬워졌다** (스프레드시트 뷰어 + 본문 마크다운 표 공통): 세로 격자선을 걷어내고 가로 구분선만 남겼고, 한 줄 걸러 아주 옅은 음영을 넣었다. **숫자 열은 오른쪽 정렬**에 자릿수 폭을 고정해 금액을 위아래로 비교할 수 있다. 파일에 없는 천단위 쉼표를 임의로 만들어 넣지는 않는다 — 엑셀은 셀에 저장된 서식이 곧 표시값이라, 뷰어가 쉼표를 그리면 파일과 다른 걸 보여주게 된다.

### Added

- **표에서 지금 보는 칸이 어디인지 보인다**: 셀에 마우스를 올리면 **그 행과 그 열이 함께** 옅게 강조된다.
- **스크롤해도 기준이 남는다**: 아래로 내려도 **제목 행**이, 옆으로 밀어도 **행 번호와 첫 열**이 제자리에 고정된다. 제목 행은 "1행이 열 이름처럼 보이는가"를 판단해서 잡는다(CSV에는 제목 행이라는 개념 자체가 없어, 무조건 첫 줄로 단정하지 않는다).

## [0.9.3] - 2026-07-20

### Added

- **CSV 파일도 엑셀처럼 표로 열린다**: 탐색기에서 `.csv`를 클릭하면 지금까지는 OS 기본 앱으로 넘어갔지만, 이제 엑셀과 **같은 뷰어**에서 표로 열린다(같은 표 렌더·확대·10,000행 고지). UTF-8 CSV는 BOM이 있어도 첫 열 제목이 깨지지 않고, 날짜 열은 `31413` 같은 내부 숫자가 아니라 `1986-01-01`로 보인다.
  - 알려진 한계: **CP949(EUC-KR)로 저장된 CSV는 글자가 깨져 보인다.** 요즘 엑셀·구글시트·카카오톡 내보내기는 모두 UTF-8이라 실사용 파일 59개를 확인했을 때 해당 사례가 하나도 없어 지원을 넣지 않았다. 깨지는 파일이 생기면 알려주면 대응한다.

## [0.9.2] - 2026-07-19

### Fixed

- **파일을 열어도 하단 브레드크럼프(폴더 경로)가 그대로이던 문제**: 뷰어는 브레드크럼프를 갱신하지 않아, 화면 전체가 그 PDF인데도 푸터는 이전 문서의 폴더를 가리키고 있었다. 이제 **브레드크럼프는 본문에 보이는 것의 폴더**를 따라간다 — 파일을 열면 그 파일의 폴더로, 닫으면 다시 보던 문서의 폴더로 돌아온다. `Esc`나 `✕`로 닫아도 마찬가지다(뷰어가 스스로 닫히는 경우도 알리도록 내부 계약을 넓혔다).

## [0.9.1] - 2026-07-19

### Changed

- **뷰어 전용 헤더 줄을 없애고 앱 상단바로 합쳤다**: 0.9.0에서 뷰어가 자기만의 헤더 줄을 하나 더 그리는 바람에 화면 한 줄을 통째로 낭비했다. 이제 **파일명은 앱 상단바에**, **확대·축소·닫기 버튼은 상단바 우측 기존 버튼들 옆에** 들어간다. 한 줄이 통째로 문서 공간으로 돌아왔다.
- **뷰어 버튼이 앱의 다른 버튼과 똑같아졌다**: 테두리·배경이 박힌 별도 디자인이었던 것을, 모드·테마·설정 버튼과 **완전히 같은 버튼 컴포넌트**로 바꿨다(납작한 아이콘, 호버에만 옅은 배경). 확대/축소도 `−`/`+` 텍스트가 아니라 앱과 같은 아이콘이고, 뷰어 버튼과 앱 버튼 사이는 `|` 선으로 나눈다. 이제 둘이 서로 달라질 수 없다 — 같은 컴포넌트를 쓰기 때문.

### Fixed

- 뷰어를 열면 상단바 좌측 아이콘들(탐색기·최근·목차·열기)이 파일명 뒤로 밀리던 문제. 아이콘 묶음이 항상 상단바 맨 앞에 남는다.

## [0.9.0] - 2026-07-18

### Changed

- **파일 뷰어가 뜨는 모달에서 본문 영역 전체를 쓰는 화면으로 바뀌었다**: PDF·한글(HWP)·이미지·엑셀·HTML을 열면 이제 작은 모달이 아니라 **에디터 자리를 그대로 차지하는 전체 화면**으로 열린다. 상단바·탐색기 사이드바·하단 푸터는 그대로 남는다. 좁은 모달에 욱여넣느라 생기던 크기 문제(잘림·과대 확대)가 구조적으로 사라졌다.
- **뷰어 상단에 헤더가 생겼다**: 왼쪽에 파일명(이미지는 해상도, 엑셀은 행 수까지), 오른쪽에 **확대/축소 `[−] 100% [+]`** 와 닫기 `✕`. 전체 화면이 되면서 파일명이 공간을 차지할 값을 하게 됐다.
- **확대 버튼이 생겼다**: 뷰어의 `[+]`/`[−]`로 내용을 키우고 줄인다(50%~300%). `100%` 라벨을 누르면 화면 맞춤으로 돌아간다. 열 때마다 화면 맞춤(100%)에서 시작한다. **이 확대는 에디터 본문 글자 크기(⌘+/−)와 완전히 분리**돼 있어, 에디터를 키워놨다고 뷰어가 덩달아 커지지 않는다(0.8.6에서 잡은 문제의 구조적 마무리).
- **Esc 또는 ✕를 누르면 보던 문서로 정확히 돌아간다** — 문서 내용·스크롤 위치·커서가 그대로 보존된다(에디터를 내리지 않고 잠시 숨기는 방식). 뷰어를 열어둔 채 탐색기에서 마크다운 문서를 클릭하면 뷰어가 알아서 닫히고 그 문서가 열린다.

## [0.8.6] - 2026-07-18

### Fixed

- **PDF·HWP 페이지가 본문보다 2배 크게 열려 내용이 잘려 보이던 문제(실제 원인 수정)**: 두 뷰어가 페이지 크기를 계산할 때 **에디터 본문 글자 배율(⌘+/−로 키운 값)까지 곱하고** 있었다. 그래서 에디터를 1.5배로 키워 두면 PDF·HWP도 1.5배로 열려 패널 밖으로 넘치고 좌우가 잘렸다. 이제 두 뷰어는 **에디터 확대와 무관하게** 항상 페이지 전체가 패널 폭의 90%로 들어오게 맞춘다(양옆 여백 유지). 0.8.5의 "90%"가 실제로는 에디터 배율에 곱해져 커지던 것을 바로잡았다. (참고: 뷰어 안에서 ⌘+/−로 확대하는 기능은 이번에 분리됐다 — ⌘+/−는 에디터 본문에만 적용된다.)

### Changed

(0.8.1~0.8.3의 PDF 수정이 겨냥하던 진짜 원인이 이번에 잡혔다 — 개발/골든 환경은 물론 Playwright WebKit에서도 재현되지 않고 오직 실제 앱 웹뷰에서만 나타나던 버그라, 실제 빌드 번들로 원인을 확정해 고쳤다.)

## [0.8.0] - 2026-07-16

### Added

- **PDF 파일 뷰어**: 탐색기에서 `.pdf` 파일을 클릭하면 앱 안에서 바로 열린다. 페이지는 스크롤해 내려갈 때 필요한 만큼만 그려서 문서가 길어도 여는 즉시 뜨고, 글자는 드래그해 선택·복사할 수 있다. ⌘±로 확대·축소된다.
- **뷰어 켜기/끄기 설정**: 설정 › 뷰어에서 파일 형식별(이미지·한글·엑셀·HTML·PDF)로 앱 내부 뷰어를 켜고 끌 수 있다. 끈 형식은 탐색기에서 클릭하면 앱 대신 운영체제의 기본 프로그램으로 열린다. 기본값은 전부 켜짐이라 평소에는 신경 쓸 필요가 없다.

## [0.7.0] - 2026-07-15

### Added

- **한글(HWP·HWPX) 파일 뷰어**: 탐색기에서 `.hwp`·`.hwpx` 파일을 클릭하면 앱 안에서 바로 열린다. 표·2단 조판·삽화·밑줄 같은 서식이 원본 그대로 보이며, 페이지는 스크롤해 내려갈 때마다 필요한 만큼만 그린다(문서가 수백 페이지여도 여는 즉시 뜬다). 읽기 전용 미리보기라 텍스트 선택·편집은 되지 않는다.
- **HTML 파일 뷰어**: 탐색기에서 `.html`·`.htm` 파일을 클릭하면 앱 안에서 렌더된다. **HTML 안의 스크립트는 실행되지 않는다** — 어디서 받았는지 모를 파일을 열어도 안전하도록 정적 렌더만 한다. 같은 폴더의 이미지는 함께 표시되지만, 별도 CSS 파일(`<link>`)은 적용되지 않는다.

### Fixed

- **탐색기에서 파일을 열 때 드물게 나던 콘솔 오류**를 제거했다.

---

## [0.6.0] - 2026-07-14

### Added

- **엑셀 파일 뷰어**: 탐색기에서 `.xlsx`·`.xls` 파일을 클릭하면 앱 안에서 바로 열린다. 시트가 여러 개면 상단 탭으로 넘길 수 있고, 셀에 보이던 서식(날짜·백분율·통화 등)이 그대로 표시된다. 값만 보여주며 수식을 다시 계산하지는 않는다. 아주 큰 시트는 10,000행까지만 그리고, 그 경우 아래에 "전체 N행 중 10,000행 표시"라고 알려준다 — 말없이 자르지 않는다.
- **제목 글꼴 설정** (설정 › 타이포그래피 › "제목 글꼴"): 헤딩에 쓸 글꼴을 고를 수 있다. 테마 기본 / **Paperlogy**(한글 헤딩용, 앱에 포함) / Georgia(세리프) 3종. 기본값인 "테마 기본"은 지금까지의 모양 그대로다.

- **사이드바 풀하이트 레일**: 파일 탐색기·목차·최근 사이드바를 창 맨 위부터 맨 아래까지 꽉 찬 세로 레일로 바꿨다. 이제 상단 헤더(탐색기·리더·설정)와 하단 상태 표시줄이 사이드바를 위아래로 가두지 않고 에디터 영역 쪽에만 놓여, 투톤 대비가 끊김 없이 이어진다.
- **심리스 크롬**: 헤더와 상태 표시줄이 에디터와 같은 캔버스색이 되고 가는 경계선만 남았다. 화면의 톤 대비는 사이드바 레일 하나가 전담해, 창 전체가 한 장의 종이처럼 읽힌다.
- **아이콘 툴바 + 사이드바로 이동**: 헤더 버튼이 전부 아이콘으로 간결해졌다(이름은 툴팁으로). 탐색기·최근·목차·경로 열기 버튼은 사이드바가 열리면 사이드바 맨 위로 옮겨 앉고, 닫으면 헤더로 돌아온다.
- **디테일 다듬기**: 사이드바 경계의 두꺼운 회색 분할선이 평소엔 보이지 않게 바뀌고(끌 때만 표시), 앱을 켜자마자 탐색기에 떠 있던 주황 테두리가 키보드 탐색 중에만 나타나도록 고쳤다. 크롬 전반의 여백을 한 기준으로 정렬하고, 선택된 파일 행에 색 막대 표시를 더했다.

## [0.5.1] - 2026-07-11

### Changed

- **탐색기 투톤 대비**: 파일 탐색기 사이드바를 에디터 본문과 뚜렷이 대비되는 투톤으로 바꿨다. 라이트·claude 테마는 어두운 사이드바, 다크 테마는 밝은 사이드바로 반전되며, 사이드바 전용 색 팔레트 덕에 어느 테마에서도 파일명·아이콘이 또렷하게 읽힌다.
- **탐색기 텍스트 줌 연동**: `⌘ +`/`⌘ -`/`⌘ 0` 줌이 이제 에디터 본문뿐 아니라 탐색기의 글자·아이콘까지 함께 확대·축소한다.

### Fixed

- **코드 블록 안 드래그 선택이 안 보이던 문제**: 코드 블록 안의 텍스트를 마우스로 끌어 선택할 때 선택 배경이 코드 블록 배경에 가려 보이지 않던 문제를 고쳤다. 모든 테마에서 선택 영역이 드러난다.
- **하이라이트 안의 서식이 적용되지 않던 문제**: `==하이라이트==` 안에 쓴 굵게·기울임·인라인 코드·링크가 서식 없이 밋밋하게 표시되던 문제를 고쳐, 하이라이트와 안쪽 서식이 함께 적용되도록 했다.

## [0.4.0] - 2026-06-23

`0.1.0`의 읽기 전용 골격에서, Obsidian식 라이브 프리뷰 에디터로 성장한 누적 릴리스다.
이 구간에는 별도 태그가 없어 0.1.0 이후의 모든 변경을 한 섹션으로 묶었다.

### Added

- **라이브 프리뷰 파이프라인**: 트리(Lezer) 기반에 커서/선택 위치를 인식하는 Obsidian식 라이브 프리뷰. 마크다운 문법 마커는 숨기고(conceal) 캐럿이 들어오면 원문을 드러낸다(reveal). 기능별 모듈을 등록하는 plugin registry 구조로, inline 데코레이션과 block 위젯을 함께 다룬다. block 데코레이션은 CM6 제약에 맞춰 StateField로 분리.
- **Mermaid 다이어그램**: 펜스 코드 블록을 SVG 위젯으로 렌더(오류 시 폴백 표시). CSS transform 기반 드래그 팬·Ctrl/Cmd-휠 줌·더블클릭 토글 지원, 다이어그램을 자연 크기로 표시하고 줌/팬을 되돌리는 플로팅 리셋 버튼, 펜스 fold(접기) 제공.
- **수식·코드·표**: KaTeX 인라인/블록 수식, 스타일 적용된 펜스 코드 블록, GFM 표 그리드, 작업 목록 체크박스 렌더.
- **콜아웃·구분선·하이라이트**: `> [!type]` 블록쿼트 콜아웃 박스 — Obsidian 표준 13종 + 별칭, 타입별 Lucide 아이콘·색, `> [!type] 제목` 커스텀 제목(생략 시 타입명). `---` 가로 구분선, `==mark==` 인라인 하이라이트(GFM 확장).
- **풋노트**: 참조는 위첨자로, 정의는 흐리게 렌더. 참조↔정의 양방향 클릭 내비게이션과 ⌘-hover 정의 프리뷰 지원.
- **위키링크·이미지**: `[[` 입력 시 파일 피커/자동완성, 대상 존재 시 활성 링크(새 창으로 열기)·없는 링크/asset 링크 처리. 로컬·원격 이미지를 asset protocol로 표시(선택적 title 파싱).
- **비디오/유튜브 임베드**: `![[URL]]`·`![이름](URL)`에서 유튜브 링크는 썸네일 facade(클릭 시 youtube-nocookie iframe 재생), 비디오 파일(mp4/webm/ogg/ogv/mov/m4v)은 `<video>` 태그로 임베드.
- **붙여넣기 자동 링크화**: 텍스트를 선택한 채 URL을 붙여넣으면 `[선택텍스트](URL)`로 감싼다(Obsidian식). 선택이 없거나 클립보드가 URL이 아니면 일반 붙여넣기.
- **에디터 기능**: edit/read 모드 토글(설정 영속), 헤딩/리스트 항목 fold(Obsidian식), Workflowy식 리스트 불릿과 fold halo, 선택 영역을 `=`·`*` 마크로 감싸기, 괄호 자동 닫기, 본문 줌(⌘ +/-/0), Vim 모드(CM6 Vim 확장), 세션 상태(스크롤·커서 위치) 영속화.
- **UI 크롬**: 하단 상태 바(Lucide 아이콘 + shadcn/Raycast 스타일 버튼), 테마 적용 스크롤바·fold 거터.
- **설정(SSOT)**: 의존성 없는 SSOT 설정 프리미티브 위에 theme·mode·vimMode를 선언, 싱크가 구독하는 단일 출처 구조. 설정 패널과 theme-as-JSON, 요소별 색을 라이브 프리뷰로 편집하는 전체 테마 비주얼 에디터(스워치 그리드 + 접이식 고급 JSON 에디터).
- **타이포그래피/테마**: OS 추종 + 토글 테마, ElevenLabs DESIGN.md 토큰 기반 타입 시스템, 헤딩 스케일·링크 대비·줌 연동 애니메이션 체크박스, Pretendard 번들 + Google Fonts 로더, 읽기 measure/tracking 조정.
- **번들 CLI**: `mermark bundle` CLI와 `bundle_doc` 커맨드로 LLM 컨텍스트 패키징(부모/절대 위키링크 추종), ⌘⇧C로 LLM 컨텍스트 번들을 클립보드에 복사.
- **CLI/창 제어**: 단일 파일 인자 열기, 누락된 파일 인자 자동 생성(Vim식), `--right`로 화면 우측 절반에 창 띄우기.

### Changed

- 펜스 코드를 block 위젯으로 렌더하도록 전환하고, 코드 본문 정의를 한 곳으로 통합. 위키링크로 생성된 창에도 capability 부여.
