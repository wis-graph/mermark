# mermark 폴리시 마스터플랜 (재귀 수렴 → 최단경로)

> `recursive-refinement-plan` 산출물. **새 기능 없이** 기존 기능의 사용성·디테일만 6렌즈로 재귀 폴리시 → 수렴 → 최단경로 종합. **계획 문서 — 구현은 별도.**
> 기준 버전: v0.4.0 · 작성: 2026-06-24

---

## 품질 기준 (6렌즈 위의 톤)
**볼트 무게 없이 즉각·일관·예측가능한 Obsidian급 편집 품질.** 다듬어졌다 = 기능을 우연히 발견하고, 한 동작으로 닿고, 결과가 즉시 보이며, 다른 기능과 똑같이 행동하고, 이상한 입력에도 안 깨진다.

6렌즈: ①발견성 ②마찰 ③피드백 ④일관성 ⑤엣지케이스 ⑥접근성·키보드.

## 마스터플랜 (최단경로)

> 정렬 원칙: 체감 빈도÷비용. **일관성 항목은 묶어서 한 번에**(흩으면 또 어긋남).

### P1 — 신뢰·고빈도 마찰 (매 세션 체감)
**R1 저장 상태 표시 · R2 임베드 실패 피드백 · R6 paste 커서 · R7 conflict 흐름**
- 왜 먼저: 매 편집 세션에서 닿는 신뢰·피드백 갭. 저장이 조용한 건 데이터 에디터에서 가장 큰 불안. 저비용·고체감.

### P2 — 일관성 통일 (배치, 한 번에)
**R5 위젯 reveal·alt-click·cursor 통일 + R11 상태바 정보 + R12 위젯 인터랙션 언어 + R14 vim 상태 + R16 모션 타이밍**
- 왜 묶나: 일관성은 흩어 고치면 다음 기능에서 또 어긋난다. 위젯 동작·상태바·모션을 **한 라운드에 한 톤으로** 정의해야 "예측가능"이 산다. 인터랙션 언어 SSOT 한 번.

### P3 — 발견성 (한 번 배우면 평생)
**R3 단축키·제스처 발견(도움말 오버레이 + affordance) · R4 콜아웃 타입 자동완성**
- 왜: 숨은 기능(⌘hover·alt-click·번들·풋노트nav)이 많다. 발견성은 1회 투자로 전 기능의 체감 가치를 올린다.

### P4 — 견고성·접근성 (낮은 빈도, 높은 완성도)
**R9 다크/라이트 전수 · R15 aria·키보드 도달 · R10 엣지(거대/빈/중복/중첩) · R8 mermaid 인터랙션 · R13 설정/테마 에디터 사용성**
- 왜 마지막: 빈도는 낮지만 "안 깨진다·누구나 쓴다"가 완성도의 바닥. P1~P3가 일상 체감을 올린 뒤 바닥을 채운다.

### 곁가지 (새 기능 — `recursive-improvement-plan` 소관, 폴리시 아님)
- **커맨드 팔레트**(진짜 ⌘P 액션 실행) — R3의 "도움말 오버레이"(기존 단축키 안내)와 구분: 팔레트는 새 기능.
- **위키링크 hover 프리뷰**(풋노트엔 있으나 위키링크엔 없음) — 일관성 욕구가 가리키지만 새 위젯 = 새 기능.
- **문서 내 검색** — 폴리시가 아니라 신규.
→ 전부 IMPROVEMENT 마스터플랜으로 넘김.
