# Welcome Screen Design Specification

## Goal
파일 경로 없이 앱이 실행되었을 때 빈 임시 문서를 강제로 띄우는 대신, 중앙 정렬된 1컬럼 형태의 웰컴 화면을 제공하여 사용자가 직접 문서를 선택해 작업을 시작할 수 있도록 돕습니다. 또한 최초 실행 시 홈 디렉토리(`~`)와 문서 디렉토리(`~/Documents`)를 즐겨찾기에 기본 등록합니다.

## Requirements

### 1. 기본 즐겨찾기 자동 등록
- 최초 실행 시(로컬 스토리지에 즐겨찾기 키 `mermark.favoriteFolders`가 전혀 없을 때) 사용자의 홈 디렉토리(`~`)와 `~/Documents` 디렉토리 경로를 자동으로 구하여 즐겨찾기에 등록합니다.
- `@tauri-apps/api/path` 플러그인의 `homeDir()` 및 `documentDir()` API를 호출하여 절대 경로를 획득합니다.
- Tauri 권한 설정(`capabilities/default.json`)에 `"path:default"`를 추가하여 API 접근 권한을 확보합니다.

### 2. 웰컴 화면 레이아웃 (우측 에디터 영역)
- 파일 경로(`file` 파라미터) 없이 `index.html`이 단독 호출되면, 에디터 호스트 영역 대신 **웰컴 화면(Welcome Pane)**을 렌더링합니다.
- **레이아웃**: 1컬럼 세로 흐름, 전체 중앙 정렬, 브라우저 너비에 맞춘 자연스러운 여백 확보.
- **구성 요소**:
  - **상단 섹션**: `즐겨찾기` 폴더 목록
    - 등록된 홈 폴더 및 Documents 폴더 등이 단정한 행(Row) 혹은 카드 형태로 노출됩니다.
    - 클릭 시, 좌측 사이드바가 파일 탐색기(Explorer) 모드로 활성화되면서 해당 폴더 경로로 갱신(점프)되어 폴더 내의 파일을 선택할 수 있게 합니다.
  - **하단 섹션**: `최근 문서` 파일 목록
    - 최근 열어본 파일명 및 전체 경로가 심플하게 노출됩니다.
    - 클릭 시 해당 문서를 즉시 에디터에 로드합니다.

## UI / CSS Design
- `.welcome-pane` 컨테이너는 flex-column으로 중앙 정렬 및 여백을 부드럽게 확보합니다.
- 각 리스트의 행(Row)들은 마우스 호버 시 은은한 배경색 변화 효과를 주어 인터랙션을 제공합니다.
