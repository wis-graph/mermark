# 설정 패널 컴포넌트 스펙 (디자인 제외)

> mermark 설정 패널에 들어가는 **컴포넌트 요소의 기능·구조·동작 계약**만 기술한다. 색·간격·타이포·모션 등 **비주얼 디자인은 의도적으로 배제** — 그 결정은 디자이너(클로드디자인)에게 위임하며, 이 문서는 "무엇이 존재하고 어떻게 동작해야 하는가"의 불변 계약이다. 디자인 리뉴얼이 어떤 비주얼을 택하든 **아래 계약은 보존되어야 한다**.
>
> 기준 코드: `src/settings/panel/modal.ts`(셸), `src/settings/panel/controls.ts`(컨트롤), `src/settings/app.ts`(항목 선언), `src/settings/store.ts`·`registry.ts`(프리미티브). 스타일: `src/styles.css:662-887`.

---

## 1. 패널 셸 (Shell)

2-pane 모달. DOM 계층:

```
.settings-backdrop                  ← 전체 화면 백드롭(클릭 시 닫힘)
  .settings-modal                   ← 모달 컨테이너 (flex column)
    .settings-header                ← 타이틀 + 닫기(X) 버튼
    .settings-body                  ← flex row
      .settings-sidebar             ← 카테고리 버튼 세로 목록
      .settings-pane                ← 선택된 카테고리의 설정 행들
```

**기능 계약 (디자인 무관하게 유지):**
- **Lazy build**: 패널 DOM은 상태바 `⚙ 설정` 버튼 **첫 클릭** 시에만 생성. 부팅 시엔 버튼만(콜드로드 비용 0).
- **열기**: 상태바 설정 버튼 클릭. 열 때 에디터에 `inert` 부여, 첫 카테고리 버튼에 포커스.
- **닫기 3경로**: ESC 키 / 백드롭 클릭 / 헤더 X 버튼. 닫을 때 `inert` 해제 + 직전 포커스(`lastFocused`) 복원.
- **포커스 트랩**: Tab이 모달 내부에서만 순환(양끝 wrap). 모달 밖으로 새지 않음.
- **카테고리 전환**: 사이드바 버튼 클릭 → pane 내용 교체(`selectCategory`). 한 번에 한 카테고리만 표시.
- **구독 정리(teardown)**: pane 교체·모달 닫기 시 이전 컨트롤들의 setting 구독 해제(누수 방지).

## 2. 카테고리 (Sidebar)

현재 5개. **삽입(등록) 순서**가 곧 표시 순서이며 첫 등장이 위치를 고정한다(`registry.ts groups()`).

| 순서 | 카테고리 | 비고 |
|------|----------|------|
| 1 | 테마 | 프리셋 + 비주얼 테마 에디터 |
| 2 | 타이포그래피 | 글꼴·크기·너비·줄간격·제목비율 |
| 3 | 에디터 | 모드·자동저장·충돌·Vim·이미지검색 |
| 4 | Mermaid | 팬줌·다이어그램 테마 |
| 5 | 플러그인 | (플레이스홀더 — info만) |

**계약**: 새 그룹은 새 `group` 문자열을 쓰면 자동 생성. 그룹 내 항목도 등록 순. 디자인은 카테고리 구분/서브헤딩을 자유롭게 추가 가능하나 **순서와 그룹 멤버십은 데이터(`ui.group`)가 결정**한다.

## 3. 행 셸 (Row)

모든 설정 항목은 공통 셸을 공유한다(ad-hoc 아님 — `controls.ts row()`):

```
.settings-row
  .settings-row-label      ← 항목 라벨 (텍스트)
  .settings-row-control    ← 컨트롤 위젯 (아래 6종 중 1)
```

**계약**: 라벨 + 컨트롤 1:1. (현재 라벨 셀 124px 고정폭이 긴 라벨을 자르는 약점 — 디자인이 해소 대상.)

## 4. 컨트롤 위젯 6종 (동작 계약)

`kind`로 디스패치되는 렌더러 테이블(`controls.ts RENDER`). **모든 컨트롤의 공통 라운드트립 계약**:
1. 마운트 시 `setting.get()` 값을 반영.
2. 사용자 입력 시 `setting.set(newValue)`.
3. `setting.subscribe(reflect)`로 **외부 변경**(다른 경로의 set, 예: ⌘E·⌘±·상태바 토글)도 컨트롤 UI에 즉시 반영.

| kind | 용도 | 값 타입 | 동작 계약 | 현재 사용 항목 |
|------|------|---------|-----------|----------------|
| **segmented** | 2~3개 상호배타 선택 | string(1:1) | pill 버튼 그룹, 1개만 active | 프리셋·기본모드·충돌정책·Vim·이미지검색·팬줌·다이어그램테마 (8) |
| **select** | N개 중 1개(드롭다운) | string(1:1) | `<select>` + `<option>`, value=setting값 | 글꼴·제목비율 (2) |
| **slider** | 연속 수치 | number | min/max/step 범위, **현재값 라이브 표시**(단위 포함) | 본문크기·너비·줄간격·자동저장지연 (4) |
| **text** | 자유 문자열 | string | 텍스트 입력, 선택적 도움말 텍스트 | 웹폰트 (1) |
| **json** | 테마 비주얼 에디터 | Theme 객체 | §5 별도 명세 | 테마 JSON (1) |
| **info** | 정적 안내 | — | 읽기 전용 텍스트(컨트롤 없음) | 플러그인 플레이스홀더 (1) |

**디자인 메모**: 현재 16항목 중 8개가 segmented라 시각적으로 단조롭다. 디자이너는 segmented/select/slider/text를 Claude 디자인 언어로 재가공하되, **kind별 동작 계약(상호배타/1:1/범위+라이브값/자유입력)은 불변**.

## 5. 테마 비주얼 에디터 (`json` 컨트롤 — 가장 복잡)

`renderJson()` (`controls.ts:183-306`). 두 부분:

### 5a. 스워치 그리드
- **18장 컬러 카드** = 코어 9 + 마크다운 9(`CORE_CARDS`/`MARKDOWN_CARDS`). 각 카드 = `<input type="color">`(색 선택기) + 라벨.
- 마크다운 카드(h1~h6/bold/italic/code/highlight)는 **라이브 프리뷰**: 선택한 색이 해당 CSS var에 즉시 바인딩돼 카드 라벨 글자색이 실시간 변함.
- 색 변경 → `setting.set({...activeTheme, colors:{...,[key]:value}})`. **프리셋 이름(`name`) 보존**.

### 5b. JSON 직접 편집 (아코디언)
- `<details>` 접이식 안에 `<textarea>`(테마 JSON 전문) + **적용·복사·내려받기** 버튼.
- **적용**: `parseTheme()` strict 검증 → 실패 시 인라인 에러 메시지, 통과 시 `setting.set`.
- **내려받기**: Blob으로 `theme.json` 파일 저장.

**계약**: 18 스워치 + 라이브 프리뷰 + JSON 아코디언(검증/적용/복사/내려받기)은 **기능적으로 모두 유지**. 디자인은 레이아웃(그리드 열 수·카드 모양·아코디언 비주얼)을 자유롭게 재구성 가능.

## 7. 리뉴얼이 보존해야 할 불변 계약 (체크리스트)

디자인이 어떻게 바뀌든 아래는 깨지면 안 된다:

- [ ] Lazy build(첫 클릭 생성), 포커스 트랩, ESC/백드롭/X 3-경로 닫기, inert·포커스 복원.
- [ ] 모든 컨트롤의 `get→반영` / `input→set` / `subscribe→외부변경 반영` 라운드트립.
- [ ] 카테고리·항목 **등록 순서** = 표시 순서(데이터 주도).
- [ ] 테마 에디터: 18 스워치 + 마크다운 카드 라이브 프리뷰 + JSON 검증/적용/복사/내려받기.
- [ ] kind별 동작 의미(segmented=상호배타, select=N중1, slider=범위+라이브값, text=자유, json=테마, info=정적).
- [ ] **줌 가드**: 패널은 `.cm-content`/`.cm-line` 밖(에디터 측정 트리 무접촉) — 이미 충족, 유지.
- [ ] **테마 인식**: 패널 자체가 `--bg`/`--fg`/`--surface`/`--border`/`--accent` 등 테마 var를 쓰므로 다크/라이트/클로드 어디서도 일관돼야 함(하드코딩 색 금지).

## 8. 디자인 위임 범위 (이 문서가 다루지 않는 것)

색·간격·타이포·라운드·모션·컨트롤의 시각적 형태·아이콘·서브헤딩·행 위계 — **전부 디자이너 결정**. 비주얼 언어는 `DESIGN_CLAUDE.md` + 동반 요청서(`docs/SETTINGS_REDESIGN_BRIEF.md`) 참조.
